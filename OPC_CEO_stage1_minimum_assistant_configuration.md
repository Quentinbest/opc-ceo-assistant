# 一人公司（OPC）CEO 阶段一最小助理配置详解

下面展开的是一人公司 OPC 在**阶段一：刚起步**时的“最小助理配置”。

这个阶段的目标不是把公司做复杂，而是先做到：

> **能获客、能成交、能交付、能收钱、账清楚、合同安全、CEO 不被琐事拖垮。**

所以阶段一最小配置建议是：

1. **财务/税务助理**
2. **法务助理**
3. **销售助理**
4. **CEO 办公室助理**
5. **项目/运营助理**

这五类助理可以组成一人公司的最小经营中枢。

---

## 一、阶段一最小配置总览

| 助理 | 核心目标 | 主要防止什么问题 | 推荐配置方式 |
|---|---|---|---|
| 财务/税务助理 | 管钱、管账、管税 | 账乱、税务风险、现金流断裂 | 外包会计 + AI 财务助理 + 表格 |
| 法务助理 | 管合同、管风险 | 合同漏洞、收款纠纷、知识产权风险 | 律师按次审查 + AI 合同初审 |
| 销售助理 | 管客户、管线索、管成交 | 客户丢失、跟进混乱、报价随意 | CRM 表格 + AI 销售文案 |
| CEO 办公室助理 | 管日程、任务、文档、提醒 | CEO 忘事、乱忙、没有重点 | AI 助理 + 日程/任务工具 |
| 项目/运营助理 | 管交付、流程、异常 | 成交后交付失控、客户不满意 | SOP + 项目看板 + 兼职/AI |

## 二、财务/税务助理

### 1. 为什么阶段一必须有财务/税务助理？

一人公司刚起步时，CEO 最容易犯的错误是：

> 只关心有没有收入，不关心账是否清楚、税是否合规、钱是否真的够用。

但公司真正危险的不是“没有利润”，而是：

- 不知道自己赚没赚钱；
- 不知道客户还欠多少钱；
- 不知道下个月现金够不够；
- 发票、报税、合同金额对不上；
- 私账、公账混用；
- 成本、费用、利润完全混乱；
- 业务赚钱，但现金流断裂。

所以财务/税务助理是阶段一的第一刚需。

### 2. 财务/税务助理的核心职责

#### A. 收入管理

负责记录每一笔收入。

| 项目 | 说明 |
|---|---|
| 客户名称 | 谁付的钱 |
| 收入类型 | 产品收入、服务收入、咨询收入、订阅收入等 |
| 合同金额 | 合同约定金额 |
| 已收金额 | 实际到账金额 |
| 未收金额 | 还欠多少钱 |
| 收款日期 | 实际到账时间 |
| 开票状态 | 未开票、已开票、无需开票 |
| 回款备注 | 是否延期、是否分期、是否有争议 |

CEO 必须随时知道：

> 谁欠我钱？欠多少？什么时候该收？有没有逾期？

#### B. 成本费用管理

负责记录公司花出去的钱。

| 类型 | 示例 |
|---|---|
| 直接成本 | 采购成本、外包成本、项目成本、物流成本 |
| 软件订阅 | ChatGPT、Claude、Notion、飞书、服务器、域名 |
| 市场费用 | 广告、设计、剪辑、推广 |
| 办公费用 | 电脑、打印、耗材、快递 |
| 差旅费用 | 交通、住宿、餐饮 |
| 专业服务费 | 会计、律师、顾问 |
| 税费 | 增值税、所得税、附加税等 |

阶段一尤其要分清：

> **成本**和**费用**不是一回事。

例如：

- 为某个客户项目雇外包，这是项目成本；
- 每月固定的软件订阅，是运营费用；
- 公司成立、记账、审合同，是专业服务费。

#### C. 现金流管理

现金流是阶段一最重要的财务指标。

利润表上赚钱，不等于账户里有钱。

比如：

```text
本月签了 10 万合同
但客户 30 天后付款
你本月还要支付 4 万成本
那你现在可能依然缺钱
```

财务助理要帮助 CEO 看：

| 指标 | CEO 需要知道什么 |
|---|---|
| 当前现金余额 | 现在账户里还有多少钱 |
| 未来 30 天应收 | 最近一个月能收多少钱 |
| 未来 30 天应付 | 最近一个月必须付多少钱 |
| 固定支出 | 每月不管有没有收入都要花多少钱 |
| 现金流缺口 | 是否会出现没钱支付的情况 |
| 安全现金储备 | 至少要留几个月生存资金 |

阶段一建议至少保留：

> **3 个月固定支出现金储备。**

#### D. 税务合规

税务助理负责提醒：

- 什么收入需要申报；
- 什么发票需要保留；
- 哪些费用可以入账；
- 哪些票据不合规；
- 什么时候报税；
- 有哪些小微企业优惠政策；
- 不同合同开票方式对应什么税务影响。

一人公司千万不要把税务当成“年底再处理”的事情。

税务应该从第一笔收入开始规范。

### 3. 财务/税务助理的典型产出

| 产出物 | 用途 |
|---|---|
| 收入明细表 | 看每一笔收入来源 |
| 成本费用表 | 看钱花在哪里 |
| 应收账款表 | 看谁还没付款 |
| 应付账款表 | 看自己还欠谁钱 |
| 现金流预测表 | 看未来是否缺钱 |
| 月度利润简表 | 看本月是否赚钱 |
| 发票台账 | 管理开票、收票、认证 |
| 税务事项日历 | 防止错过申报时间 |
| 月度财务简报 | 帮 CEO 做经营判断 |

### 4. 阶段一推荐配置

不要一开始招全职财务。

推荐组合是：

```text
外包会计 / 代理记账
+
AI 财务分析助理
+
标准化财务表格
+
CEO 每周查看一次
```

#### 外包会计负责

- 记账；
- 报税；
- 发票处理；
- 税务申报；
- 基础合规。

#### AI 财务助理负责

- 分类整理费用；
- 生成财务摘要；
- 分析现金流风险；
- 解释财务数据；
- 给 CEO 准备问题清单。

#### CEO 自己必须掌握

- 当前现金余额；
- 本月收入；
- 本月成本；
- 本月利润；
- 应收账款；
- 应付账款；
- 税务风险。

### 5. 财务/税务助理每周工作流

```text
每周一：
整理上周收入、支出、应收、应付

每周三：
检查客户回款、供应商付款、发票状态

每周五：
生成一页财务简报

每月底：
汇总月度收入、成本、费用、利润、现金流

每月报税前：
与会计确认发票、收入、费用、申报数据
```

### 6. 财务/税务助理检查清单

| 检查项 | 是否完成 |
|---|---|
| 每一笔收入是否记录？ |  |
| 每一笔支出是否有凭证？ |  |
| 客户欠款是否有跟进？ |  |
| 本月是否知道利润？ |  |
| 本月是否知道现金余额？ |  |
| 是否有税务申报日历？ |  |
| 发票是否与合同金额一致？ |  |
| 公账和私账是否分开？ |  |
| 是否保留合同、发票、付款记录？ |  |

## 三、法务助理

### 1. 为什么阶段一必须有法务助理？

阶段一最大的法务风险通常不是复杂诉讼，而是最基础的商业规则没有写清楚。

常见问题包括：

- 客户付款条件不明确；
- 项目交付标准不明确；
- 客户要求无限修改；
- 没有验收机制；
- 没有违约条款；
- 没有知识产权归属；
- 没有保密条款；
- 没有证据保存意识；
- 外包人员交付成果归属不清。

一人公司不能等到出事后才找律师。

法务助理的核心价值是：

> **在签合同前降低风险，而不是出纠纷后补救。**

### 2. 法务助理的核心职责

#### A. 合同起草

阶段一常用合同包括：

| 合同类型 | 适用场景 |
|---|---|
| 服务合同 | 咨询、设计、开发、运营、顾问服务 |
| 销售合同 | 产品、商品、设备、系统销售 |
| 采购合同 | 向供应商采购商品或服务 |
| 外包合同 | 找外部人员做设计、开发、剪辑、写作等 |
| 保密协议 | 涉及客户资料、商业计划、技术资料 |
| 合作协议 | 与渠道、合伙人、代理商合作 |
| 授权协议 | 品牌、内容、代码、图片、软件授权 |

阶段一可以先准备 3 个核心模板：

1. **客户服务合同模板**
2. **供应商/外包合同模板**
3. **保密协议模板**

#### B. 合同审查

法务助理要重点检查这些条款：

| 条款 | 重点检查 |
|---|---|
| 主体信息 | 对方公司名称、统一社会信用代码、联系人是否正确 |
| 服务范围 | 到底做什么，不做什么 |
| 交付标准 | 什么叫完成 |
| 交付时间 | 何时交付，延期怎么办 |
| 验收方式 | 客户几天内验收，不回复是否视为通过 |
| 付款节点 | 预付款、尾款、分期付款 |
| 发票条款 | 是否开票，税点如何约定 |
| 违约责任 | 谁违约，怎么赔 |
| 退款条款 | 什么情况下可退，什么情况下不可退 |
| 修改次数 | 客户能免费修改几次 |
| 知识产权 | 成果归谁，何时转让 |
| 保密义务 | 双方保密范围和期限 |
| 争议解决 | 出纠纷在哪里解决 |

#### C. 证据管理

很多纠纷不是输在合同，而是输在没有证据。

法务助理要帮助 CEO 保存：

| 证据类型 | 示例 |
|---|---|
| 合同文件 | 纸质合同、电子合同、扫描件 |
| 沟通记录 | 微信、邮件、会议纪要、聊天截图 |
| 付款凭证 | 银行流水、转账截图、收据 |
| 发票记录 | 开票申请、已开发票、收票记录 |
| 交付记录 | 文件链接、交付邮件、系统截图 |
| 验收记录 | 客户确认、签收、邮件回复 |
| 修改记录 | 客户提出的变更需求 |
| 催款记录 | 催款邮件、短信、聊天记录 |

一人公司要建立一个原则：

> **所有关键业务沟通，都要留下可追溯记录。**

### 3. 法务助理的典型产出

| 产出物 | 用途 |
|---|---|
| 合同模板库 | 提高签约效率 |
| 合同审查清单 | 防止漏看风险条款 |
| 风险条款说明 | 告诉 CEO 哪些地方不能让步 |
| 客户合同档案 | 方便后续查找 |
| 供应商合同档案 | 管理外部合作风险 |
| 催款函模板 | 用于回款异常 |
| 纠纷证据包 | 发生争议时快速响应 |
| 法务风险周报 | 提醒即将发生的合同风险 |

### 4. 阶段一推荐配置

不要一开始请全职法务。

推荐组合是：

```text
AI 合同草拟/初审
+
CEO 明确商业条件
+
律师按次审查关键合同
+
合同档案系统
```

#### AI 适合做

- 起草合同初稿；
- 提取合同关键条款；
- 生成风险清单；
- 对比两个合同版本；
- 准备给律师的问题；
- 整理谈判要点。

#### 律师适合做

- 审查重要合同；
- 修改风险条款；
- 处理纠纷；
- 出具律师函；
- 设计标准合同体系；
- 判断具体法律后果。

#### CEO 必须亲自决定

- 价格；
- 付款方式；
- 交付范围；
- 能否退款；
- 是否接受延期；
- 哪些条款可以让步；
- 哪些条款绝不让步。

### 5. 法务助理工作流

```text
客户有合作意向
↓
销售助理整理商业条件
↓
法务助理选择合同模板
↓
AI 生成合同初稿或审查对方合同
↓
CEO 确认商业条款
↓
重要合同交给律师审查
↓
双方签署
↓
合同归档
↓
项目交付过程中保存证据
↓
收款完成后归档结案
```

### 6. 法务助理检查清单

| 检查项 | 是否完成 |
|---|---|
| 合同主体是否正确？ |  |
| 付款节点是否明确？ |  |
| 交付范围是否明确？ |  |
| 验收标准是否明确？ |  |
| 是否限制免费修改次数？ |  |
| 是否写明逾期付款责任？ |  |
| 是否约定知识产权归属？ |  |
| 是否约定保密义务？ |  |
| 是否保留关键沟通证据？ |  |
| 是否归档合同和发票？ |  |

## 四、销售助理

### 1. 为什么阶段一必须有销售助理？

阶段一公司最重要的问题是：

> 有没有客户？客户能不能成交？成交后能不能收钱？

一人公司 CEO 通常亲自销售，但不能完全靠脑子记客户。

否则会出现：

- 聊过的客户忘记跟进；
- 报价不统一；
- 客户需求记录不清；
- 成交概率无法判断；
- 不知道哪个渠道有效；
- 客户流失了也不知道原因；
- 该催款时没有及时催；
- 低价值客户占用大量时间。

销售助理的核心作用是：

> **把客户机会从“聊天记录”变成“销售管道”。**

### 2. 销售助理的核心职责

#### A. 客户线索管理

每个潜在客户都要进入线索表。

| 字段 | 说明 |
|---|---|
| 客户名称 | 公司或个人名称 |
| 联系人 | 关键联系人 |
| 联系方式 | 电话、微信、邮箱 |
| 来源渠道 | 朋友介绍、社媒、广告、官网、线下 |
| 客户需求 | 客户想解决什么问题 |
| 预算范围 | 客户能接受的价格 |
| 决策人 | 谁最终拍板 |
| 当前阶段 | 初聊、需求确认、报价、谈判、成交、流失 |
| 下次跟进时间 | 防止忘记 |
| 成交概率 | 高、中、低 |
| 预计金额 | 可能成交金额 |
| 备注 | 客户顾虑、特殊情况 |

#### B. 客户跟进

销售助理要提醒 CEO：

- 今天该联系谁；
- 哪些客户超过 3 天没跟进；
- 哪些客户看了报价但没回复；
- 哪些客户快到付款节点；
- 哪些客户需要补充资料；
- 哪些客户应该放弃。

一人公司销售要避免两个极端：

1. **追得太紧，让客户反感；**
2. **跟得太松，让客户流失。**

所以销售助理需要建立跟进节奏。

例如：

```text
初次沟通后 1 天：发送整理后的需求确认
报价后 2 天：询问是否有疑问
报价后 5 天：提供案例或补充方案
报价后 10 天：确认是否继续推进
超过 30 天无反馈：进入低优先级线索池
```

#### C. 报价管理

阶段一报价不能太随意。

销售助理要帮助 CEO 准备：

| 报价材料 | 用途 |
|---|---|
| 标准报价单 | 避免每次从零报价 |
| 套餐说明 | 让客户容易选择 |
| 服务范围说明 | 防止客户误解 |
| 交付时间说明 | 管理客户预期 |
| 不包含事项 | 防止无限加需求 |
| 付款方式 | 预付款、阶段款、尾款 |
| 有效期 | 防止报价长期无效 |
| 常见问题 | 降低重复解释成本 |

报价最好包含一句边界说明：

> 本报价仅包含以上列明服务内容，超出范围的新增需求将另行评估费用与交付周期。

#### D. 销售材料准备

阶段一至少要有一套基础销售材料。

| 材料 | 作用 |
|---|---|
| 公司/个人介绍 | 让客户知道你是谁 |
| 服务说明 | 让客户知道你能做什么 |
| 案例展示 | 建立信任 |
| 报价单 | 推进成交 |
| FAQ | 回答常见顾虑 |
| 合作流程 | 降低客户决策成本 |
| 客户证言 | 增强可信度 |
| 付款说明 | 明确收款方式 |

### 3. 销售助理的典型产出

| 产出物 | 用途 |
|---|---|
| 客户线索表 | 管理所有潜在客户 |
| 销售管道看板 | 看每个客户处于哪个阶段 |
| 每日跟进清单 | 提醒 CEO 联系客户 |
| 报价单模板 | 提高报价效率 |
| 客户需求记录 | 防止沟通失真 |
| 流失原因表 | 分析为什么没成交 |
| 成交复盘表 | 总结为什么成交 |
| 客户分层表 | 区分高价值客户和低价值客户 |

### 4. 阶段一推荐配置

```text
CEO 亲自销售
+
AI 销售助理
+
CRM 表格
+
标准报价模板
```

#### AI 销售助理可以做

- 根据客户描述整理需求；

## 五、CEO 办公室助理

### 1. 为什么阶段一需要 CEO 办公室助理？

一人公司最大的问题是：

> CEO 同时是老板、销售、客服、财务、项目经理、运营、行政。

这会导致 CEO 进入“碎片化忙碌”状态：

- 每天都很忙，但不知道忙了什么；
- 重要客户忘记跟进；
- 合同忘记归档；
- 发票忘记处理；
- 项目进度没人追；
- 会议结束没有行动项；
- 想法很多，但没有执行清单；
- 一直救火，没时间做增长。

CEO 办公室助理的核心作用是：

> **把 CEO 的注意力从杂事中解放出来，让 CEO 专注于决策、销售、产品和关键关系。**

### 2. CEO 办公室助理的核心职责

#### A. 日程管理

负责管理 CEO 的时间。

| 工作 | 说明 |
|---|---|
| 安排会议 | 客户沟通、供应商会议、项目会议 |
| 设置提醒 | 合同、付款、交付、报税、续费 |
| 避免冲突 | 防止时间重叠 |
| 预留深度工作时间 | 用于产品、战略、内容、销售 |
| 周计划 | 明确本周最重要的 3 件事 |
| 日计划 | 明确今天必须完成的任务 |

#### B. 任务管理

CEO 办公室助理要把所有事项分成几类：

| 类型 | 示例 |
|---|---|
| 今日必须做 | 今天必须完成，否则有损失 |
| 本周重点 | 本周推动业务增长的关键任务 |
| 客户跟进 | 与客户相关的行动项 |
| 项目交付 | 与正在执行项目相关的任务 |
| 财务事项 | 收款、付款、发票、报税 |
| 法务事项 | 合同、协议、证据归档 |
| 待决策事项 | 需要 CEO 做判断的问题 |
| 可延后事项 | 不紧急、不重要的事情 |

阶段一最有效的任务管理原则是：

> 每天最多抓 3 个关键任务。

#### C. 会议与沟通管理

一人公司虽然人少，但仍然有很多沟通：

- 客户沟通；
- 供应商沟通；
- 外包人员沟通；
- 会计沟通；
- 律师沟通；
- 合作伙伴沟通。

办公室助理要帮助 CEO 形成：

| 文档 | 作用 |
|---|---|
| 会议议程 | 会前明确讨论什么 |
| 会议纪要 | 会后记录决定了什么 |
| 行动项清单 | 谁在什么时候做什么 |
| 决策记录 | 为什么做这个决定 |
| 跟进提醒 | 到期提醒 CEO 追进度 |

#### D. 文档归档

阶段一文档不多，但必须从一开始建立秩序。

建议文件夹结构如下：

```text
公司资料
├── 01_证照资质
├── 02_财务税务
├── 03_客户合同
├── 04_供应商合同
├── 05_销售报价
├── 06_项目交付
├── 07_发票付款
├── 08_市场内容
├── 09_SOP流程
└── 10_复盘总结
```

| 产出物 | 用途 |

| 每周重点计划 | 防止忙乱 |

## 六、项目/运营助理

### 1. 为什么阶段一需要项目/运营助理？

销售成交只是开始，真正决定公司口碑的是交付。

阶段一常见问题是：

- 客户成交后没人管流程；
- 交付标准不清楚；
- 客户临时加需求；
- 项目延期；
- CEO 忘记某个交付节点；
- 外包人员交付质量不稳定；
- 客户验收迟迟不确认；
- 项目完成后没有复盘；
- 每次都靠 CEO 临场发挥。

项目/运营助理的核心作用是：

> **把每一单业务从“靠人记”变成“靠流程跑”。**

### 2. 项目/运营助理的核心职责

#### A. 项目启动

项目一旦成交，必须从销售状态进入交付状态。

项目启动时要明确：

| 内容 | 说明 |
|---|---|
| 项目目标 | 最终要交付什么 |
| 交付物 | 文件、产品、服务、报告、系统、商品 |
| 交付时间 | 每个节点什么时候完成 |
| 客户联系人 | 谁负责确认 |
| 内部负责人 | 一人公司通常是 CEO |
| 外部协作者 | 设计、开发、供应商、物流等 |
| 验收标准 | 什么叫完成 |
| 风险点 | 哪些地方可能延期或出问题 |

#### B. 任务拆解

项目助理要把一个项目拆成任务。

例如一个服务项目：

```text
签约
↓
收预付款
↓
收集客户资料
↓
确认需求
↓
制定方案
↓
第一次交付
↓
客户反馈
↓
修改完善
↓
最终交付
↓
客户验收
↓
收尾款
↓
开票
↓
归档
↓
复盘
```

每个任务都要有：

- 负责人；
- 截止时间；
- 输入资料；
- 输出成果；
- 完成标准。

#### C. 交付质量管理

项目/运营助理要帮助 CEO 检查：

| 检查项 | 说明 |
|---|---|
| 是否符合合同范围 | 没有少做，也没有无偿多做 |
| 是否符合客户需求 | 解决客户真正的问题 |
| 是否达到验收标准 | 可以让客户确认完成 |
| 是否按时交付 | 防止延期 |
| 是否有交付记录 | 方便证明已经完成 |
| 是否有客户确认 | 防止后续争议 |
| 是否保存最终版本 | 方便归档和复用 |

#### D. 异常处理

阶段一一定会遇到异常。

常见异常包括：

| 异常 | 处理方式 |
|---|---|
| 客户临时加需求 | 判断是否属于合同范围，超出范围则重新报价 |
| 客户迟迟不反馈 | 发送提醒，并设置默认确认机制 |
| 外包人员延期 | 启用备用方案或重新排期 |
| 交付质量不达标 | 返工、替换协作者、降低损失 |
| 客户拖欠尾款 | 销售助理 + 财务助理 + 法务助理共同处理 |
| 项目范围失控 | 回到合同和需求确认文档 |

### 3. 项目/运营助理的典型产出

| 产出物 | 用途 |
|---|---|
| 项目启动表 | 明确项目基本信息 |
| 项目任务看板 | 跟踪进度 |

| 验收标准表 | 防止争议 |

## 十一、阶段一最小配置的优先级

如果时间有限，按这个顺序做。

### 第一优先级：财务/税务助理

因为不管有没有业务，公司都要面对钱、账、税。

必须先做：

- 收入表；
- 支出表；
- 应收账款表；
- 发票记录；
- 报税日历；
- 找会计。

### 第二优先级：销售助理

因为没有客户，公司无法生存。

必须先做：

- 客户线索表；
- 跟进提醒；
- 报价模板；
- 服务介绍；
- 销售阶段管理。

### 第三优先级：法务助理

因为客户一旦成交，就要签合同。

必须先做：

- 合同模板；
- 合同审查清单；
- 付款条款；
- 交付条款；
- 验收条款；
- 证据归档。

### 第四优先级：项目/运营助理

因为成交后必须交付。

必须先做：

- 项目启动表；
- 交付清单；
- 项目看板；
- 验收标准；
- 复盘模板。

### 第五优先级：CEO 办公室助理

因为 CEO 需要一个总控台。

必须先做：

- 每日任务清单；
- 每周计划；
- 重要提醒；
- 文件归档；
- 决策记录；
- 周报模板。

## 十二、最终形成的阶段一最小工作流

最小配置可以总结为一条经营主线：

```text
销售助理找到客户
↓
CEO 判断是否值得做
↓
销售助理整理需求和报价
↓
法务助理准备合同
↓
财务/税务助理确认收款和发票
↓
客户签约付款
↓
项目/运营助理组织交付
↓
CEO 完成关键工作
↓
客户验收
↓
财务/税务助理收尾款、开票、记账
↓
CEO 办公室助理归档、提醒、复盘
```

一句话总结：

> 阶段一的一人公司 CEO，不需要先招很多人，而要先建立五个最小助理系统：**财务/税务管钱，法务管风险，销售管客户，项目/运营管交付，CEO 办公室管秩序。**  
> 这五个系统搭起来，公司就能从“CEO 一个人凭记忆硬撑”，变成“有流程、有记录、有提醒、有复盘地经营”。
