# 第一阶段 OPC CEO 助理的 Agent Skill 化方案

第一阶段的一人公司（OPC）CEO 助理体系非常适合构建成 **Agent Skill**。

但关键不是把每个助理都做成“完全自主行动的 Agent”，而是把它们拆成可调用、可复用、可审计的 **Agent Skill**。

换句话说：

> **助理是角色，Skill 是能力模块，Agent 是执行主体。**

---

## 一、结论：第一阶段助理非常适合 Skill 化

第一阶段的一人公司 CEO 面临的问题高度重复、流程清晰、风险可控，非常适合构建 Agent Skill。

主要原因有五个：

| 原因 | 说明 |
|---|---|
| 任务重复 | 每周都要整理财务、客户、合同、项目、周报 |
| 输入明确 | 合同、聊天记录、报价单、发票、客户需求、项目进度 |
| 输出稳定 | 清单、表格、摘要、风险提示、邮件草稿、周报 |
| 流程可标准化 | 可以沉淀为 SOP、模板、检查清单 |
| 需要人类最终判断 | 正好适合“AI 准备材料，CEO 决策”模式 |

所以第一阶段最适合构建的不是“一个万能 CEO Agent”，而是：

```text
CEO Orchestrator Agent
│
├── 财务/税务 Skill
├── 法务合同 Skill
├── 销售跟进 Skill
├── CEO 办公室 Skill
├── 项目/运营 Skill
├── 周报复盘 Skill
└── 文档归档 Skill
```

---

## 二、先区分：Agent、Assistant、Skill 的关系

### 1. Assistant：角色

例如：

- 财务助理
- 法务助理
- 销售助理
- 项目助理
- CEO 办公室助理

它们回答的是：

> **谁来帮 CEO 做这类事情？**

---

### 2. Skill：能力包

例如：

- 合同审查 Skill
- 现金流分析 Skill
- 客户跟进 Skill
- 报价生成 Skill
- 项目拆解 Skill
- 经营周报 Skill

它们回答的是：

> **这件事如何标准化地完成？**

---

### 3. Agent：执行者

例如：

- CEO Orchestrator Agent
- Finance Agent
- Legal Agent
- Sales Agent
- Project Agent

它们回答的是：

> **谁来调用这些 Skill，并根据上下文推进任务？**

---

### 4. 最合理关系

对于一人公司阶段一，最佳结构是：

```text
CEO
│
└── CEO Orchestrator Agent
    │
    ├── 调用财务/税务 Skill
    ├── 调用法务 Skill
    ├── 调用销售 Skill
    ├── 调用项目/运营 Skill
    ├── 调用办公室 Skill
    └── 生成总结、提醒、建议
```

不要一开始就做五个复杂 Agent。

更合理的是：

> **先做五组 Skill，由一个总控 Agent 调用。**

---

## 三、哪些助理最适合构建成 Agent Skill？

### 总体适配度

| 助理 | 是否适合 Skill 化 | 适合程度 | 原因 |
|---|---:|---:|---|
| 财务/税务助理 | 适合 | 极高 | 数据结构清晰，输出稳定 |
| 法务助理 | 适合 | 极高 | 合同审查、风险清单高度模板化 |
| 销售助理 | 适合 | 极高 | 客户跟进、报价、话术可流程化 |
| CEO 办公室助理 | 适合 | 极高 | 日程、任务、周报、提醒高度重复 |
| 项目/运营助理 | 适合 | 极高 | 项目拆解、SOP、验收清单可标准化 |

结论：

> 阶段一五类助理都适合 Skill 化。  
> 但它们适合 Skill 化的部分不同，必须拆成具体技能，而不是笼统地做成“财务助理 Agent”或“法务助理 Agent”。

---

## 四、财务/税务助理适合构建哪些 Agent Skill？

财务/税务助理非常适合 Skill 化，但要注意边界：

> AI 可以做整理、分析、提示和问题清单；  
> 但不能替代会计和税务顾问做最终合规判断。

---

### 1. 收入支出整理 Skill

#### 适用场景

CEO 上传或粘贴：

- 银行流水；
- 支付记录；
- 发票记录；
- Excel 表格；
- 手工记账文本。

#### Skill 任务

把杂乱记录整理成结构化表格。

#### 输入

```text
日期、金额、客户/供应商、摘要、付款方式、备注
```

#### 输出

| 输出 | 说明 |
|---|---|
| 收入明细表 | 每笔收入分类 |
| 支出明细表 | 每笔支出分类 |
| 待确认项目 | 无法判断类别的记录 |
| 异常提醒 | 金额异常、重复记录、缺少凭证 |
| 汇总摘要 | 本周/本月收入支出概况 |

---

### 2. 现金流预测 Skill

#### 适用场景

CEO 想知道：

> 下个月钱够不够？

#### 输入

- 当前现金余额；
- 应收账款；
- 应付账款；
- 固定支出；
- 预计收入；
- 预计税费。

#### 输出

| 输出 | 说明 |
|---|---|
| 未来 30 天现金流预测 | 看短期资金安全 |
| 未来 90 天现金流预测 | 看阶段性风险 |
| 现金流缺口 | 什么时候可能缺钱 |
| 风险等级 | 低、中、高 |
| 行动建议 | 催款、延迟付款、削减支出等 |

---

### 3. 应收账款跟进 Skill

#### 适用场景

客户已经成交，但还没付款。

#### 输入

- 客户名称；
- 合同金额；
- 已收金额；
- 未收金额；
- 付款节点；
- 当前逾期天数；
- 历史沟通记录。

#### 输出

| 输出 | 说明 |
|---|---|
| 应收账款清单 | 谁欠钱、欠多少 |
| 催款优先级 | 哪些客户最该先催 |
| 催款话术 | 温和版、正式版、强硬版 |
| 风险提示 | 是否可能演变成纠纷 |
| 下一步动作 | 什么时候再次跟进 |

---

### 4. 月度财务简报 Skill

#### 适用场景

每月底自动生成经营财务摘要。

#### 输出

```text
本月收入：
本月支出：
本月利润：
当前现金余额：
应收账款：
应付账款：
现金流风险：
异常费用：
下月建议：
```

#### 价值

让 CEO 每月最少看一次公司的真实经营状况。

---

### 5. 财务/税务 Skill 边界

| 可以做 | 不应该做 |
|---|---|
| 整理流水 | 替代正式记账 |
| 分析现金流 | 替代会计判断 |
| 提醒税务事项 | 替代税务申报 |
| 生成问题清单 | 直接给出确定性税务结论 |
| 辅助发票分类 | 判断复杂税务合规 |

---

## 五、法务助理适合构建哪些 Agent Skill？

法务助理非常适合构建 Skill，尤其适合做：

- 合同审查；
- 风险条款识别；
- 合同摘要；
- 谈判要点；
- 证据归档清单。

但也必须有边界：

> AI 法务 Skill 负责“发现风险、解释风险、准备材料”，不负责替代律师作最终法律意见。

---

### 1. 合同审查 Skill

#### 适用场景

CEO 收到客户或供应商合同，想快速知道有没有坑。

#### 输入

- 合同全文；
- 交易背景；
- 业务类型；
- 金额；
- 付款方式；
- 交付内容；
- CEO 最关心的问题。

#### 输出

| 输出 | 说明 |
|---|---|
| 合同摘要 | 这份合同到底约定了什么 |
| 关键条款表 | 付款、交付、验收、违约、知识产权 |
| 高风险条款 | 对公司明显不利的地方 |
| 缺失条款 | 合同里应该有但没有的内容 |
| 修改建议 | 建议如何改 |
| 律师问题清单 | 需要律师确认的问题 |

---

### 2. 合同模板生成 Skill

#### 适用场景

CEO 要与客户签服务合同、销售合同、外包合同。

#### 输入

```text
业务类型：
客户是谁：
服务内容：
价格：
付款节点：
交付时间：
验收方式：
是否开票：
是否包含知识产权：
是否允许退款：
```

#### 输出

- 合同初稿；
- 关键条款说明；
- 可协商条款；
- 不建议让步条款；
- 律师审查提示。

---

### 3. 合同风险对比 Skill

#### 适用场景

客户修改了合同，CEO 想知道改动有什么风险。

#### 输入

- 原合同版本；
- 修改后合同版本。

#### 输出

| 输出 | 说明 |
|---|---|
| 条款差异 | 哪些地方被改了 |
| 风险变化 | 风险变大还是变小 |
| 对 CEO 的影响 | 钱、交付、责任、知识产权 |
| 建议回应 | 接受、拒绝、重新谈判 |

---

### 4. 证据归档 Skill

#### 适用场景

项目交付、客户验收、催款、纠纷前准备材料。

#### 输出

```text
需要保存的证据：
1. 合同版本
2. 报价单
3. 付款记录
4. 沟通记录
5. 交付记录
6. 验收确认
7. 催款记录
8. 发票记录
```

#### 价值

提前为可能的争议保留证据链。

---

## 六、销售助理适合构建哪些 Agent Skill？

销售助理是最适合 Skill 化的角色之一。

原因是销售流程天然就是管道：

```text
线索 → 初聊 → 需求确认 → 报价 → 谈判 → 成交 → 回款 → 复购
```

---

### 1. 客户线索整理 Skill

#### 输入

客户聊天记录、电话纪要、微信内容、邮件内容。

#### 输出

| 输出 | 说明 |
|---|---|
| 客户基本信息 | 公司、联系人、联系方式 |
| 客户需求 | 客户真正想解决什么 |
| 痛点 | 为什么现在要解决 |
| 预算 | 大概愿意花多少钱 |
| 决策人 | 谁能拍板 |
| 成交概率 | 高、中、低 |
| 下一步动作 | 应该怎么跟进 |

---

### 2. 报价方案生成 Skill

#### 输入

- 客户需求；
- 服务范围；
- 交付周期；
- 成本估算；
- 利润目标；
- 客户预算。

#### 输出

- 基础版报价；
- 标准版报价；
- 高级版报价；
- 每个版本包含内容；
- 不包含事项；
- 付款方式；
- 报价有效期；
- 风险提醒。

---

### 3. 客户跟进 Skill

#### 输入

- 上次沟通内容；
- 客户状态；
- 报价时间；
- 客户顾虑；
- 跟进目标。

#### 输出

| 版本 | 用途 |
|---|---|
| 温和提醒版 | 客户刚看完报价 |
| 价值补充版 | 客户还在犹豫 |
| 截止日期版 | 报价快过期 |
| 唤醒沉默客户版 | 客户很久没回复 |
| 放弃线索版 | 礼貌结束跟进 |

---

### 4. 成交/流失复盘 Skill

#### 输入

- 客户来源；
- 沟通记录；
- 报价；
- 成交结果；
- 客户反馈。

#### 输出

```text
成交原因：
流失原因：
客户真实顾虑：
报价是否合理：
销售话术问题：
下次优化建议：
是否值得继续跟进：
```

---

## 七、CEO 办公室助理适合构建哪些 Agent Skill？

CEO 办公室助理最适合做成“总控型 Skill”。

它不一定深入某个专业领域，但负责：

- 任务管理；
- 提醒；
- 会议纪要；
- 每日计划；
- 每周复盘；
- 文件归档；
- 决策记录。

---

### 1. 每日 CEO Briefing Skill

#### 适用场景

每天早上让 CEO 快速进入工作状态。

#### 输入

- 日程；
- 待办；
- 客户跟进；
- 项目进度；
- 财务提醒；
- 合同提醒。

#### 输出

```text
今日最重要的 3 件事：
今日客户跟进：
今日项目交付：
今日财务事项：
今日合同事项：
今日风险提醒：
建议优先级：
```

---

### 2. 每周经营周报 Skill

#### 输入

- 客户线索表；
- 项目进度表；
- 收入支出表；
- 应收账款表；
- 合同台账；
- CEO 本周记录。

#### 输出

```text
本周销售情况：
本周收入支出：
本周项目进度：
本周合同风险：
本周客户反馈：
本周 CEO 关键决策：
下周最重要的 3 件事：
需要 CEO 亲自处理的事项：
```

---

### 3. 会议纪要 Skill

#### 输入

会议录音转文字、会议笔记、聊天记录。

#### 输出

| 输出 | 说明 |
|---|---|
| 会议摘要 | 讨论了什么 |
| 决策事项 | 已经决定什么 |
| 待办事项 | 谁负责，什么时候完成 |
| 风险点 | 有哪些不确定事项 |
| 后续跟进 | 下一次要做什么 |

---

### 4. 决策记录 Skill

#### 适用场景

CEO 做了重要决定，比如：

- 是否接某个客户；
- 是否降价；
- 是否更换供应商；
- 是否延期交付；
- 是否聘请外包。

#### 输出

```text
决策事项：
背景：
可选方案：
选择方案：
选择理由：
预期收益：
主要风险：
观察指标：
复盘时间：
```

这个 Skill 很重要，因为它能帮助 CEO 避免“凭感觉经营”。

## 八、项目/运营助理适合构建哪些 Agent Skill？

项目/运营类任务非常适合 Skill 化，因为它本质上是：

> 把承诺变成流程，把流程变成交付，把交付变成验收。

---

### 1. 项目启动 Skill

#### 输入

- 合同；
- 报价单；
- 客户需求；
- 销售承诺；
- 交付时间；
- 预算；
- 外包资源。

#### 输出

```text
项目名称：
项目目标：
交付物：
里程碑：
任务拆解：
负责人：
截止时间：
客户需提供资料：
验收标准：
风险点：
```

---

### 2. 项目任务拆解 Skill

#### 输入

项目目标和交付物。

#### 输出

| 输出 | 说明 |
|---|---|
| 阶段拆解 | 项目分几个阶段 |
| 任务列表 | 每阶段做什么 |
| 输入资料 | 开始任务前需要什么 |
| 输出成果 | 完成后交付什么 |
| 时间节点 | 每项任务什么时候完成 |
| 验收标准 | 什么叫完成 |
| 风险提醒 | 哪些地方可能卡住 |

---

### 3. 客户反馈整理 Skill

#### 输入

客户反馈、聊天记录、邮件、批注。

#### 输出

```text
反馈分类：
1. 必须修改
2. 可选修改
3. 超出合同范围
4. 需要客户澄清
5. 建议另行报价

下一步建议：
```

这个 Skill 特别重要，因为它能防止客户无限加需求。

---

### 4. 项目验收 Skill

#### 输入

- 合同；
- 交付清单；
- 客户反馈；
- 最终交付物；
- 验收标准。

#### 输出

```text
已完成交付物：
未完成事项：
客户需确认事项：
验收邮件草稿：
尾款提醒：
归档清单：
复盘问题：
```

## 九、最推荐的阶段一 Agent Skill 架构

### 1. 不建议的架构

不建议一开始这样做：

```text
财务 Agent
法务 Agent
销售 Agent
项目 Agent
办公室 Agent
```

原因是：

- 太重；
- 容易互相重复；
- 编排复杂；
- 权限边界难控制；
- 一人公司初期没有那么多数据；
- CEO 反而要管理一堆 Agent。

---

### 2. 推荐的架构

建议这样做：

```text
CEO Orchestrator Agent
│
├── Skill 01：每日 CEO Briefing
├── Skill 02：客户线索整理
├── Skill 03：报价方案生成
├── Skill 04：合同审查
├── Skill 05：合同模板生成
├── Skill 06：项目启动
├── Skill 07：项目任务拆解
├── Skill 08：客户反馈整理
├── Skill 09：应收账款跟进
├── Skill 10：现金流预测
├── Skill 11：经营周报
└── Skill 12：文档归档
```

这个架构更符合一人公司的现实：

> 一个总控 Agent，多个专业 Skill，CEO 保留最终决策权。

## 十、阶段一最值得优先构建的 12 个 Agent Skill

### 优先级总表

| 优先级 | Skill | 所属助理 | 价值 |
|---:|---|---|---|
| 1 | 每日 CEO Briefing Skill | CEO 办公室 | 防止 CEO 忘事和乱忙 |
| 2 | 客户线索整理 Skill | 销售 | 防止客户丢失 |
| 3 | 报价方案生成 Skill | 销售 | 提高成交效率 |
| 4 | 合同审查 Skill | 法务 | 降低签约风险 |
| 5 | 项目启动 Skill | 项目/运营 | 防止成交后交付失控 |
| 6 | 项目任务拆解 Skill | 项目/运营 | 让交付可执行 |
| 7 | 应收账款跟进 Skill | 财务/税务 | 防止钱收不回来 |
| 8 | 现金流预测 Skill | 财务/税务 | 防止现金流断裂 |
| 9 | 客户反馈整理 Skill | 项目/运营 | 防止需求失控 |
| 10 | 会议纪要 Skill | CEO 办公室 | 防止沟通无结果 |
| 11 | 经营周报 Skill | CEO 办公室 | 让 CEO 每周复盘经营 |
| 12 | 文档归档 Skill | CEO 办公室/法务 | 防止资料丢失 |

## 十一、每个 Agent Skill 应该如何设计？

一个合格的 Agent Skill 不只是一个提示词，而应该包含以下结构：

```text
Skill 名称
Skill 目标
适用场景
输入要求
执行步骤
输出格式
检查清单
风险边界
是否需要 CEO 审批
是否需要专家确认
示例输入
示例输出
```

---

### 标准 Skill 模板

```markdown
# Skill 名称

## 1. 目标

说明这个 Skill 要解决什么问题。

## 2. 适用场景

说明什么时候调用这个 Skill。

## 3. 输入

列出用户需要提供的信息。

## 4. 执行步骤

按顺序说明 Agent 应该怎么处理。

## 5. 输出格式

规定输出必须是什么结构。

## 6. 检查清单

列出输出前必须检查的事项。

## 7. 风险边界

说明哪些事情不能由 AI 最终决定。

## 8. 需要人工审批的情况

说明什么时候必须交给 CEO、会计、律师或其他专家。

## 9. 示例

提供一个完整示例。
```

## 十二、示例：合同审查 Agent Skill

```markdown
# 合同审查 Skill

## 1. 目标

帮助 CEO 快速识别客户合同、供应商合同、外包合同中的商业风险和法律风险，并生成修改建议和律师问题清单。

## 2. 适用场景

当 CEO 收到一份合同，准备签署前调用本 Skill。

## 3. 输入

- 合同全文
- 交易背景
- 合同金额
- 服务/商品内容
- 付款方式
- 交付周期
- CEO 最关心的问题

## 4. 执行步骤

1. 提取合同基本信息。
2. 总结合同核心义务。
3. 检查付款条款。
4. 检查交付和验收条款。
5. 检查违约责任。
6. 检查退款、延期、终止条款。
7. 检查知识产权和保密条款。
8. 标记高风险条款。
9. 给出修改建议。
10. 生成需要律师确认的问题。

## 5. 输出格式

- 合同摘要
- 关键条款表
- 高风险条款
- 缺失条款
- 修改建议
- CEO 决策提醒
- 律师问题清单

## 6. 检查清单

- 是否明确付款节点？
- 是否明确验收标准？
- 是否限制免费修改次数？
- 是否约定逾期付款责任？
- 是否约定知识产权归属？
- 是否有保密条款？
- 是否有争议解决条款？

## 7. 风险边界

本 Skill 只能做合同风险识别和修改建议，不能替代律师出具正式法律意见。

## 8. 需要人工审批的情况

以下情况必须交给律师：

- 合同金额较大；
- 涉及知识产权转让；
- 涉及长期独家合作；
- 对方要求高额违约责任；
- 涉及员工、股权、投资、融资；
- CEO 无法判断条款后果。
```

## 十三、示例：每日 CEO Briefing Skill

```markdown
# 每日 CEO Briefing Skill

## 1. 目标

帮助一人公司 CEO 在每天开始工作前，快速明确今日重点、客户跟进、项目交付、财务事项和风险提醒。

## 2. 适用场景

每天早上启动工作时调用。

## 3. 输入

- 今日日程
- 未完成任务
- 客户线索表
- 项目进度表
- 应收账款表
- 合同台账
- CEO 临时备注

## 4. 执行步骤

1. 汇总今日所有任务。
2. 识别必须今天完成的事项。
3. 按销售、项目、财务、法务、行政分类。
4. 找出逾期或高风险事项。
5. 挑选今日最重要的 3 件事。
6. 生成时间安排建议。
7. 输出需要 CEO 亲自处理的事项。

## 5. 输出格式

# 今日 CEO 简报

## 今日最重要的 3 件事

1.
2.
3.

## 客户跟进

## 项目交付

## 财务事项

## 合同/法务事项

## 风险提醒

## 建议时间安排

## 今日可延后事项
```

## 十五、哪些事情不适合完全 Skill 化？

虽然这些助理适合 Skill 化，但不是所有工作都能交给 Agent Skill。

### 1. 不适合完全自动化的事项

| 事项 | 原因 |
|---|---|
| 最终报价决策 | 涉及商业判断和客户关系 |
| 是否接某个客户 | 涉及风险、战略、机会成本 |
| 是否签署合同 | 涉及法律责任 |
| 税务申报 | 需要专业会计/税务确认 |
| 法律意见 | 需要律师确认 |
| 付款审批 | 涉及资金安全 |
| 客户冲突处理 | 涉及关系和谈判 |
| 招聘和解雇 | 涉及法律和人情判断 |
| 重大供应商选择 | 涉及长期合作风险 |

---

### 2. 必须设置人工审批的节点

阶段一 Agent Skill 必须遵守：

```text
凡是涉及钱、合同、法律、税务、客户承诺、对外发送，都必须人工确认。
```

具体来说：

| 动作 | 是否需要 CEO 审批 |
|---|---|
| 生成报价 | 需要 |
| 发送报价 | 需要 |
| 起草合同 | 需要 |
| 发送合同 | 需要 |
| 催款 | 需要 |
| 修改交付范围 | 需要 |
| 承诺交付时间 | 需要 |
| 对外发送正式邮件 | 需要 |
| 删除文件 | 需要 |
| 付款 | 必须需要 |

## 十六、阶段一 Agent Skill 的推荐建设顺序

不要一口气做完所有 Skill。

建议按这个顺序建设。

---

### 第 1 组：先解决“不漏事”

1. 每日 CEO Briefing Skill
2. 会议纪要 Skill
3. 文档归档 Skill
4. 经营周报 Skill

原因：

> 先让 CEO 的工作变得有秩序。

---

### 第 2 组：再解决“能成交”

5. 客户线索整理 Skill
6. 客户跟进 Skill
7. 报价方案生成 Skill
8. 成交/流失复盘 Skill

原因：

> 一人公司第一阶段最核心的问题是获得客户和现金流。

---

### 第 3 组：再解决“能安全签约”

9. 合同审查 Skill
10. 合同模板生成 Skill
11. 合同版本对比 Skill
12. 证据归档 Skill

原因：

> 成交越多，合同风险越高。

---

### 第 4 组：再解决“能稳定交付”

13. 项目启动 Skill
14. 项目任务拆解 Skill
15. 客户反馈整理 Skill
16. 项目验收 Skill

原因：

> 成交后交付不好，会损害口碑并影响回款。

---

### 第 5 组：最后强化“现金流安全”

17. 收入支出整理 Skill
18. 应收账款跟进 Skill
19. 现金流预测 Skill
20. 月度财务简报 Skill

原因：

> 财务 Skill 越早建立越好，但自动化程度要逐步提高，避免早期数据不规范导致误判。

## 十八、最终判断

第一阶段的这些助理不仅适合构建成 Agent Skill，而且是最适合 Skill 化的一类场景。

原因是：

> 它们不是纯创意任务，而是高频、重复、流程化、需要检查清单、需要标准输出、需要人类审批的经营任务。

最优做法是：

```text
不要先做复杂 Agent
先做可复用 Skill

不要让 AI 自主决策
让 AI 准备材料、发现风险、生成草稿、提醒 CEO

不要追求全自动
先追求标准化、可审计、可复盘
```

一句话总结：

> 阶段一的五类助理，最适合被设计成一套 **OPC CEO Agent Skill Pack**：  
> 财务 Skill 管现金流，法务 Skill 管合同风险，销售 Skill 管客户机会，项目 Skill 管交付闭环，办公室 Skill 管 CEO 的任务秩序。  
> 它们共同服务一个目标：让一人公司从“CEO 凭记忆硬撑”升级为“流程化、模块化、可复盘的经营系统”。
